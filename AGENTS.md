## Пошаговое раскрытие списков

Все списки (`<ul>` и `<ol>`) на слайдах раскрываются **последовательно** — по одному
пункту за нажатие. Для этого используется класс `next` из ribbon.

Правило:

- **Первый `<li>` каждого списка — без класса `next`.** Он виден сразу при открытии слайда.
- **Каждому последующему `<li>` добавляется `class="next"`.**
- Правило действует для **каждого** списка отдельно, включая вложенные: у вложенного списка
  свой первый пункт идёт без `next`, остальные — с `next`.

```html
<ul>
    <li>Первый пункт — виден сразу</li>
    <li class="next">Открывается по нажатию</li>
    <li class="next">И этот тоже
        <ul>
            <li>Первый вложенный — виден вместе с родителем</li>
            <li class="next">Следующий вложенный — по нажатию</li>
        </ul>
    </li>
</ul>
```

При добавлении новых слайдов или пунктов — сразу проставляй `next` по этому правилу.

- **Элемент после списка тоже раскрывается по нажатию.** Если сразу за `<ul>`/`<ol>`
  идёт ещё контент (обычно `<p class="note">` или `<p class="lead">`, но может быть и
  любой другой элемент) — ему тоже добавляется `class="next"`, чтобы он появлялся
  после всех пунктов списка, а не сразу.

```html
<ul>
    <li>Первый пункт</li>
    <li class="next">Второй пункт</li>
</ul>
<p class="note next">Эта подпись появится после всех пунктов</p>
```

## Вёрстка контента

- **Не используем двухколоночную вёрстку (`columns two`) для смысловых списков.**
  Предпочитаем обычные списки (`<ul>` / `<ol>`) — они читаются последовательно и
  лучше работают с пошаговым раскрытием.
- Списки делаем без декоративных «линий»/рамок слева — обычные маркеры ribbon.
